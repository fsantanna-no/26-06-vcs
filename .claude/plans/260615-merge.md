# Merge Newest Freechains Paper Changes into tpd-21 Basis

## Context

Three venue versions of the same Freechains paper live under
`old/` in this project (`/x/papers/26-06-vcs/`):

| repo      | last change | main tex                       | role        |
|-----------|-------------|--------------------------------|-------------|
| tpd-21    | 2022-06     | `tpd-21.tex` (2000 ln)         | BASIS       |
| sbrc-23   | 2022-12     | `freechains/freechains.tex`    | condensation|
| sbseg-23  | 2023-09     | `freechains/freechains.tex`    | NEWEST       |

Goal: keep `tpd-21` as the basis (it is the most complete: journal
length, has the CRDT section and full Related Work) and merge in the
newest prose refinements from `sbrc-23` and especially `sbseg-23`.

The three are SEPARATE git repos with NO shared history, so this is
a manual prose/section-level merge, not a `git merge`. Diffs are
done by section, comparing text not commits.

## Strategy

1. Establish the basis: branch/copy `tpd-21.tex` as the working
   document (deliverable inside this worktree, e.g.
   `/x/papers/26-06-vcs/merge/tpd-21.tex`).
2. For each section, 3-way compare:
       tpd-21  <-  sbrc-23  <-  sbseg-23
   Take the NEWEST coherent prose, but preserve tpd-21-only depth
   (CRDT section, Related Work subsecs, longer Experiments) unless
   superseded.
3. Integrate sbseg-23's new subsections into tpd-21's
   "Reputation and Consensus Mechanism".
4. Reconcile `.bib`: union of `tpd-21.bib` +
   `sbseg-23/freechains/freechains.bib` + sbrc bib; dedupe keys.
5. Reconcile figures/images (`tpd-21/images/` vs sbseg figures).
6. Build (LaTeX) and proofread for dangling refs/citations.

## Risks / caveats

- No shared git history -> all merges are manual prose diffs;
  watch for silently dropped tpd-21 depth.
- Label/citation key collisions across the three bibs.
- Figure path and macro (`\FC`, custom commands) divergence between
  repos -> preamble must be reconciled first.
- Venue-specific formatting (IEEEtran vs SBC template) -> keep
  tpd-21's class; strip venue-only packages from merged-in chunks.
